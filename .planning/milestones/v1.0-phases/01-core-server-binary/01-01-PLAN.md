    **Step 2: Create handler.go**

    Create `handler.go` in the project root with package `main`. This file contains:

    1. **Response struct** (per D-01, D-02, D-03):
       ```go
       type Response struct {
           Port        int               `json:"port"`
           Method      string            `json:"method"`
           Path        string            `json:"path"`
           Timestamp   string            `json:"timestamp"`
           QueryParams map[string]string `json:"query_params"`
       }
       ```
       - `Port` is `int` (not string) per D-01 example: `"port": 8080`
       - `QueryParams` is `map[string]string` per D-02 example: `{"foo": "bar"}` -- use first value per key from `r.URL.Query()`
       - `Timestamp` is `string` formatted as `time.Now().UTC().Format(time.RFC3339)` per D-03

    2. **makeHandler function** that returns `http.HandlerFunc`:
       - Accept `port int` parameter (closure-based, not runtime parsing per research "Don't Hand-Roll" section)
       - Build `map[string]string` from `r.URL.Query()` by iterating keys and taking `v[0]` (first value)
       - Marshal `Response` struct with `json.Marshal`
       - If marshal error (unlikely but defensive): log with `slog.Error("marshal error", "error", err)` and return empty response
       - Log the request per D-06, D-07: `slog.Info("request", "port", port, "method", r.Method, "path", r.URL.Path, "remote", r.RemoteAddr)`
       - CRITICAL ORDER per Pitfall 2: Set `Content-Type` header BEFORE WriteHeader:
         ```go
         w.Header().Set("Content-Type", "application/json")
         w.WriteHeader(http.StatusOK)
         w.Write(body)
         ```
       - Imports needed: `encoding/json`, `log/slog`, `net/http`, `time`

    **Step 3: Create main.go**

    Create `main.go` in the project root with package `main`. This file contains the `main()` function implementing:

    1. **Logger initialization** (D-08, D-10):
       ```go
       logger := slog.New(slog.NewJSONHandler(os.Stdout, nil))
       slog.SetDefault(logger)
       ```

    2. **Port list**: `var ports = []int{80, 8080, 8181, 8081, 3000, 5000}`

    3. **net.Listen pre-flight for accurate startup banner** (recommended approach from research):
       - For each port, call `net.Listen("tcp", fmt.Sprintf(":%d", port))`
       - If listen fails AND port == 80: log with `slog.Error("bind failed (non-fatal)", "port", port, "error", err.Error())` and `continue` (per D-04, D-05)
       - If listen fails AND port != 80: log with `slog.Error("bind failed", "port", port, "error", err.Error())` and `continue`
       - If listen succeeds: store the `net.Listener` and port number in parallel slices/structs
       - Create `http.Server` per successful listener: `&http.Server{Addr: fmt.Sprintf(":%d", port), Handler: mux}` where mux has `mux.HandleFunc("/", makeHandler(port))`
       - Launch goroutine calling `srv.Serve(ln)` (NOT `srv.ListenAndServe()` since we already have the listener)
       - After all goroutines launched, track successful ports in `activePorts []int`

    4. **Startup banner** (D-09, D-10):
       ```go
       slog.Info("listening", "ports", activePorts)
       ```
       This emits AFTER the net.Listen loop, so `activePorts` only contains ports that actually bound.

    5. **SIGTERM graceful shutdown** (D-11, SRVR-03):
       ```go
       ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGTERM, os.Interrupt)
       defer stop()
       <-ctx.Done()

       shutdownCtx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
       defer cancel()

       var wg sync.WaitGroup
       for _, srv := range servers {
           wg.Add(1)
           go func(s *http.Server) {
               defer wg.Done()
               s.Shutdown(shutdownCtx)
           }(srv)
       }
       wg.Wait()
       ```

    - Imports needed: `context`, `fmt`, `log/slog`, `net`, `net/http`, `os`, `os/signal`, `sync`, `syscall`, `time`
    - Do NOT use `log.Fatal` or `os.Exit` anywhere in server goroutines (per anti-patterns in research)
    - Do NOT use `http.DefaultServeMux` -- each server gets its own `http.NewServeMux()` (per anti-patterns)

    Create `test.sh` in the project root. This is a bash script that starts the server, runs curl-based integration tests against all ports, validates JSON response shape, checks structured log output, tests SIGTERM shutdown, and reports pass/fail. Make it executable (`chmod +x test.sh`).

    The script must:

    1. **Header**: `#!/usr/bin/env bash` with `set -euo pipefail`

    2. **Build the binary**: `go build -o swiss-army-image .` (compile first to catch build errors)

    3. **Start server in background**: `./swiss-army-image > /tmp/sai-test-stdout.log 2>&1 &` and capture PID with `SERVER_PID=$!`

    4. **Wait for startup**: Sleep 1 second, then verify at least one port is listening: `curl -sf http://localhost:8080/ > /dev/null` with retry loop (max 5 attempts, 1s sleep between). If server never starts, print error and exit 1.

    5. **Cleanup trap**: `trap "kill $SERVER_PID 2>/dev/null; wait $SERVER_PID 2>/dev/null; rm -f swiss-army-image" EXIT`

    6. **Test battery** (each test prints PASS/FAIL and increments counters):

       a. **Test: HTTP 200 on all non-privileged ports** -- For each port in 8080, 8181, 8081, 3000, 5000:
          - `curl -sf -o /tmp/sai-resp.json -w "%{http_code}" http://localhost:$port/test/path`
          - Assert HTTP status code is "200"
          - Note: Skip port 80 since it requires root/CAP_NET_BIND_SERVICE

       b. **Test: JSON response body shape** -- For port 8080:
          - `curl -sf http://localhost:8080/hello?foo=bar`
          - Pipe to check that response contains all required keys: `"port"`, `"method"`, `"path"`, `"timestamp"`, `"query_params"`
          - Assert `"port":8080` is present
          - Assert `"method":"GET"` is present
          - Assert `"path":"/hello"` is present
          - Assert `"foo"` appears in query_params

       c. **Test: Content-Type header** -- For port 8080:
          - `curl -sf -I http://localhost:8080/` (HEAD request for headers)
          - Actually use: `curl -sf -D /tmp/sai-headers.txt http://localhost:8080/`
          - Assert headers contain `Content-Type: application/json`

       d. **Test: Any HTTP method works** -- Test POST, PUT, DELETE on port 8080:
          - `curl -sf -X POST http://localhost:8080/posttest` -- assert `"method":"POST"` in response
          - `curl -sf -X PUT http://localhost:8080/puttest` -- assert `"method":"PUT"` in response
          - `curl -sf -X DELETE http://localhost:8080/deltest` -- assert `"method":"DELETE"` in response

       e. **Test: Structured JSON log output** -- Read `/tmp/sai-test-stdout.log`:
          - Assert it contains a line with `"msg":"listening"` and `"ports"` (startup banner per D-09)
          - Assert it contains lines with `"msg":"request"` and `"port"` and `"method"` (per D-06, D-07)

       f. **Test: Graceful shutdown** -- Send SIGTERM and measure exit:
          - Record timestamp before: `START=$(date +%s)`
          - `kill -TERM $SERVER_PID`
          - `wait $SERVER_PID` (should complete)
          - Record timestamp after: `END=$(date +%s)`
          - Assert elapsed time is less than 5 seconds

    7. **Summary**: Print total passed/failed. Exit 0 if all pass, exit 1 if any fail.
