Create `Dockerfile` at the repo root with a two-stage build per D-01/CONT-01/CONT-02:

**Stage 1 — Builder:**
```dockerfile
FROM --platform=$BUILDPLATFORM golang:1.26-alpine AS build
WORKDIR /src
COPY go.mod ./
RUN go mod download
COPY *.go ./
ARG TARGETOS
ARG TARGETARCH
RUN CGO_ENABLED=0 GOOS=${TARGETOS} GOARCH=${TARGETARCH} \
    go build -trimpath -o /swiss-army-image .
```

Key points:
- `--platform=$BUILDPLATFORM` pins the builder to the runner's native arch (avoids QEMU emulation during compilation)
- `COPY go.mod` before source for layer caching (even though no external deps, this is correct practice)
- `ARG TARGETOS` and `ARG TARGETARCH` MUST be declared after the `FROM` line — BuildKit injects them only after declaration
- `-trimpath` strips absolute build paths (reproducibility, slightly smaller binary)
- `CGO_ENABLED=0` is mandatory — without it the binary may link glibc and fail on scratch

**Stage 2 — Final:**
```dockerfile
FROM scratch
EXPOSE 80 8080 8181 8081 3000 5000
COPY --from=build /swiss-army-image /swiss-army-image
ENTRYPOINT ["/swiss-army-image"]
```

Key points:
- `FROM scratch` — zero OS overhead, only the binary
- `EXPOSE` documents intent for all 6 ports (not functionally required but conventional)
- ENTRYPOINT MUST be exec-form (JSON array) — shell-form `ENTRYPOINT ./swiss-army-image` will fail because scratch has no `/bin/sh`
- Exec-form ensures the binary is PID 1 and receives SIGTERM directly (required for `docker stop` to complete in under 2 seconds per success criteria)
- No `USER` instruction — scratch has no `/etc/passwd`; this is standard for scratch-based HTTP-only images

Create `.github/workflows/build-push.yml` implementing the CI/CD pipeline per D-01 through D-06 and DIST-01/DIST-02.

**Triggers (per D-05, D-06):**
```yaml
on:
  push:
    branches:
      - main
    tags:
      - 'v*.*.*'
  pull_request:
    branches:
      - main
```

**Single job structure** (Claude's discretion — single job is simpler and avoids artifact passing between jobs):

```yaml
jobs:
  build-push:
    runs-on: ubuntu-latest
    steps:
```

**Steps in order:**

1. **Checkout** — `actions/checkout@v4`

2. **Docker metadata** — `docker/metadata-action@v6` with id `meta`
   - `images: nexus.cainc.com:5000/cainc/yriahi/swiss-army-image` (per D-02)
   - Tags block (per D-04):
     ```yaml
     tags: |
       type=semver,pattern={{version}}
       type=raw,value=latest,enable={{is_default_branch}}
       type=ref,event=branch
     ```
   - This produces: semver tag push -> `:v1.0.0` + `:latest`; main push -> `:main` + `:latest`; PR -> tags computed but not pushed

3. **Set up QEMU** — `docker/setup-qemu-action@v4` (required for ARM64 final image assembly even with cross-compilation)

4. **Set up Docker Buildx** — `docker/setup-buildx-action@v4`

5. **Log in to Nexus** — `docker/login-action@v4` with conditional (per D-03, D-06):
   ```yaml
   if: github.event_name != 'pull_request'
   with:
     registry: nexus.cainc.com:5000
     username: ${{ secrets.NEXUS_USERNAME }}
     password: ${{ secrets.NEXUS_PASSWORD }}
   ```
   - `registry: nexus.cainc.com:5000` is REQUIRED — omitting it defaults to Docker Hub (Pitfall 3 from research)
   - Conditional prevents login failure on PRs from forks (no access to secrets)

6. **Build and push** — `docker/build-push-action@v7` (per CONT-02):
   ```yaml
   with:
     context: .
     platforms: linux/amd64,linux/arm64
     push: ${{ github.event_name != 'pull_request' }}
     tags: ${{ steps.meta.outputs.tags }}
     labels: ${{ steps.meta.outputs.labels }}
     cache-from: type=gha
     cache-to: type=gha,mode=max
   ```
   - `platforms: linux/amd64,linux/arm64` satisfies CONT-02
   - `push` conditional satisfies D-06 (PRs build only, no push)
   - `cache-from/cache-to: type=gha` uses GitHub Actions cache for layer caching (zero-config, recommended approach)

**Workflow name:** `Build and Push`
